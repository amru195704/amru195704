# 第5章 高さ（P5-1・P5-2）— 手計算の期待値

対象: `pysrc/ch05/ch05_geoid.py` `ch05/ch05_height.py`（架空ジオイド格子）
検証方式: 手計算（バイリニア補間 → H = h − N）

## 入力と格子

- 点: 緯度 35.6810、経度 139.7670（内分比 t=0.72, s=0.36。P3-2と同じ）
- 4隅ジオイド高[m]（架空）: SW 36.00 / NW 36.40 / SE 36.20 / NE 36.90
- 重み: w_SW=0.1792, w_NW=0.4608, w_SE=0.1008, w_NE=0.2592

## ジオイド高 N（P5-1）

- N = 36.00×0.1792 + 36.40×0.4608 + 36.20×0.1008 + 36.90×0.2592
    = 6.4512 + 16.77312 + 3.64896 + 9.56448 = **36.43776 m**

## 標高 H（P5-2）

- 楕円体高 h = 50.000 m（GNSSが返す高さ）
- H = h − N = 50.000 − 36.43776 = **13.56224 m**
- 逆: h = H + N = 13.56224 + 36.43776 = 50.000（往復誤差 0）

## 照合

| 項目 | 手計算（期待値） | 出力 | 差 |
|---|---|---|---|
| ジオイド高 N [m] | 36.43776 | 36.43776 | 0 |
| 標高 H = h−N [m] | 13.56224 | 13.56224 | 0 |
| 往復（H→h） [mm] | 0 | 0.0000 | 0 |

→ ジオイド補間も第3章と同じバイリニア。高さは「引き算1つ」で標高と楕円体高を
   行き来できる。GNSSの楕円体高をそのまま標高と誤解しない——これが実装の要点。
