# P6-1 発生順の逐次適用 — 手計算の期待値

対象: `pysrc/ch06/ch06_sequential.py`（架空・誇張テストデータ／P3-2と同じ格子）
検証方式: 1ステップ目（地震A適用）を手計算で照合＋順序ズレの意味づけ

## 入力

- 出発点 P: 緯度 35.6810、経度 139.7670（内分比 t=0.72, s=0.36。P3-2と同じ）
- 重み: w_SW=0.1792, w_NW=0.4608, w_SE=0.1008, w_NE=0.2592（合計1）
- 地震A 4隅[秒]: SW(5.0,−3.0) NW(5.4,−3.2) SE(5.1,−3.1) NE(5.9,−3.6)

## 地震A の補正量（バイリニア）

- Δ緯度 = 5.0×0.1792 + 5.4×0.4608 + 5.1×0.1008 + 5.9×0.2592
      = 0.896 + 2.48832 + 0.51408 + 1.52928 = **5.42768 秒**
- Δ経度 = −3.0×0.1792 − 3.2×0.4608 − 3.1×0.1008 − 3.6×0.2592
      = −(0.5376 + 1.47456 + 0.31248 + 0.93312) = **−3.25776 秒**
- A適用後 緯度 = 35.6810 + 5.42768/3600 = **35.6825077**
- A適用後 経度 = 139.7670 − 3.25776/3600 = **139.7660951**
- 移動量 = √((5.42768×30.8875)² + (−3.25776×30.8875×cos35.68°)²)
        = √(167.65² + 81.72²) = **186.5 m**

## 照合

| 項目 | 手計算（期待値） | P6-1 出力 | 差 |
|---|---|---|---|
| A適用後 緯度 | 35.6825077 | 35.6825077 | 0 |
| A適用後 経度 | 139.7660951 | 139.7660951 | 0 |
| A の移動量 [m] | 186.5 | 186.51 | ≈0 |

## 順序が結果を変えること（可換でない）

- 発生順 A→B の最終点と、誤った順 B→A の最終点は一致しない。
  実行値のズレ = **約3533 mm（3.53 m）**。
- 理由: 各地震の補正量は「そのとき点がいるメッシュ位置」で決まる。
  Aを先に当てると点が動き、Bはその動いた位置で評価される。順を入れ替えると
  評価位置が変わるので、積み上げ結果が変わる。
  ⇒ **補正は発生順（時系列）に適用しなければならない**（誇張データなので
  ズレがメートル級に見えるが、実データでは桁が小さい）。
