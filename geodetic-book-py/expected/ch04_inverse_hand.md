# P4-1 逆変換（固定点反復）— 手計算の期待値

対象: `pysrc/ch04/ch04_inverse_iter.py`（架空・誇張テストデータ／P3-2と同じ格子）
検証方式: 手計算（不動点の理屈）＋実行の収束履歴の突き合わせ

## なぜ往復誤差が 0 になるか（手で言える期待値）

- 順変換を forward(P) = P + Δ(P) と書く（Δ は格子補間の補正量）。
- テスト点 P を順変換した点を P' = forward(P) = P + Δ(P) とする。
- 逆変換の反復は g(X) = P' − Δ(X) を繰り返す。
- **P を代入すると g(P) = P' − Δ(P) = (P + Δ(P)) − Δ(P) = P。**
  つまり元の点 P は反復写像 g の不動点。だから収束先は必ず P に一致し、
  **往復誤差は原理的に 0**（実装が正しければ丸め誤差のみ）。

## 収束の速さ（縮小率）

- g の微分は g'(X) = −Δ'(X)。Δ は 1メッシュ（緯度 1/120°）あたり
  約 0.8 秒しか変化しない ⇒ |Δ'| ≈ 0.8″/(1/120°) = 96 ″/° = 0.0267（無次元）。
- よって毎反復およそ ×0.03 以下で更新量が縮む（実行値もこの通り）。
