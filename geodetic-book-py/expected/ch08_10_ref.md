# 第8・9・10章（P8-1・P9-1・P10-1）— 照合

対象: `pysrc/ch08/ch08_verify.py` `ch09/ch09_par2sqlite.py`
`ch10/ch10_heatmap.py`（いずれも架空・誇張テストデータ／P3-2と同じ格子）
検証方式: 自己整合（往復残差の統計／DBと par の一致／面の連続性）

## P8-1 検証の型（往復残差の統計）

- メッシュ内 9×9＝81点で順→逆の往復残差を集計。

| 指標 | 値 | 期待 |
|---|---|---|
| 検証点数 | 81 | — |
| 最大残差 | 0.002547 mm | 許容1mm未満 |
| 平均残差 | 0.000342 mm | — |
| RMS 残差 | 0.000771 mm | — |
| 判定 | PASS | 全点<1mm |

- 検証は1点ではなく多数点の統計（最大・平均・RMS）で語る、という第8章の型を
  ミニチュアで再現。実書の73点検証と同じ考え方。

## P9-1 .par → SQLite

- P3-0 が生成した test_tokyo.par（9メッシュ）を SQLite 化し、主キー検索で
  4隅を引き直して par の生値と照合。

| メッシュ | Δ緯度[秒] | Δ経度[秒] | par一致 |
|---|---|---|---|
| 53394611(SW) | 12.00000 | −8.00000 | ✓ |
| 53394612(SE) | 12.20000 | −8.20000 | ✓ |
| 53394621(NW) | 12.40000 | −8.60000 | ✓ |
| 53394622(NE) | 12.80000 | −9.00000 | ✓ |

- DB化の前後で値が保存される（par とDBが一致）＝抽出パイプラインの正しさ。
  4隅は P3-2 の内蔵 CORNERS とも一致（段1の設計値と首尾一貫）。

## P10-1 ヒートマップ縮約版

- メッシュ内 5×5 セルの補正量の大きさ[m]を数値の面として出力。

```
（行=南→北, 列=西→東, 単位 m）
   439.6  443.5  447.4  451.4  455.3
   435.1  438.5  441.9  445.4  448.8
   430.5  433.5  436.4  439.4  442.3
   426.0  428.5  430.9  433.4  435.8
   421.5  423.5  425.4  427.4  429.3
最小 421.51 m / 最大 455.29 m
```

- 南西 421.5m →北東 455.3m へ滑らかに増える面。P3-2 の東京駅(t=0.72,s=0.36)の
  439.38m はこの面の内側に整合。ヒートマップの核は「面の数値化」で、色付けは
  matplotlib で写すだけ、という第10章の要点を確認。
