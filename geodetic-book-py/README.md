# 日本の測地系 実装編 — サンプルコード（geodetic-book-py）

Kindle本『日本の測地系 実装編 — 座標変換エンジンを作って国土地理院と
1mmで一致させる』（第2巻）の各章「Pythonで動かして確かめる」節で使う
サンプルコードです。

## これは何か / 何でないか

- **公開仕様の再表現**です。子午線弧長・ガウス・クリューゲル投影・バイリニア
  グリッド補正・固定点反復・ジオイド H=h−N など、教科書と国土地理院の公開
  情報に基づく計算を、動かして確かめられる形にしています。
- 製品版エンジン **GeoCoreJP のソースではありません**。アルゴリズムの考え方が
  同じなだけで、実装は独立です。
- **同梱データは「誇張した架空のテストデータ」**です（グリッド系）。実際の
  国土地理院パラメータではありません。補間や収束の効きが数字で見えるよう、
  変位量を実際の数十〜数百倍に誇張しています。各スクリプト冒頭に明記。
  解析系（第1・2章）のみ GRS80 の実定数を使い、公表値と照合します。

## 動かし方

標準の Python 3（3.9+ を想定。numpy 不要、標準ライブラリのみ）で動きます。

```bash
python3 ch03/ch03_make_test_par.py   # 先にテスト par を生成（P3-1/P9-1が使う）
python3 ch03/ch03_bilinear.py        # バイリニア補正（第3章）
python3 ch04/ch04_inverse_iter.py    # 反復逆変換（第4章）
python3 ch01/ch01_meridian_arc.py    # 子午線弧長（第1章）ほか
```

各スクリプトは自己完結（共通部はあえて同梱）で、単体で実行できます。

全スクリプトをまとめて実行するには、同ディレクトリで次を使います。

```bash
./run_all_samples.sh
```

Windows では次を使います。

```bat
run_all_samples.bat
```

どちらも先に `ch03/ch03_make_test_par.py` を実行してから、全 17 本のサンプルを順番に回します。

実行結果をログとして残す場合は、shell 版を次のようにリダイレクトします。

```bash
./run_all_samples.sh > all_log.txt
```

このリポジトリには、実行例として `all_log.txt` を置いてあります。

## 章別スクリプト

| 章 | スクリプト | 内容 |
| --- | --- | --- |
| 1 | ch01_ellipsoid / meridian_arc / degree_length / gauss_krueger | 楕円体基本量・子午線弧長・度あたり距離・GK投影 |
| 2 | ch02_xyz_blh / ch02_helmert | BLH⇔XYZ・ヘルマート7パラメータ |
| 3 | ch03_make_test_par / par_reader / bilinear | テストpar生成・読込・バイリニア補正 |
| 4 | ch04_inverse_iter | 固定点反復による逆変換 |
| 5 | ch05_geoid / ch05_height | ジオイド補間・H=h−N |
| 6 | ch06_sequential | 複数地震の発生順逐次適用 |
| 7 | ch07_boundary | 4隅欠損の方式差（CCQの背景） |
| 8 | ch08_verify | 往復残差の統計による検証 |
| 9 | ch09_par2sqlite | .par → SQLite |
| 10 | ch10_heatmap | 補正量の面の数値化（ヒートマップ縮約） |

各章の期待値・手計算メモは `expected/` にあります。

## 照合の考え方

- グリッド系: 手計算で期待値を作り、一致を確認（読者が自分で検算できる）。
- 解析系: GRS80 の公表定数・国土地理院「測量計算サイト」と照合。
